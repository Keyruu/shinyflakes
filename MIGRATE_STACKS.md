# Migrating Existing Stacks to the Stack Module

This document describes how to migrate existing quadlet stack files to use the
new `services.my.<name>.stack` module with inline container definitions.

## Reference: Migrated Example

See `nix/hosts/mentat/modules/stacks/karakeep.nix` for a fully migrated stack.

## Step-by-Step Migration

### 1. Update the let block

Remove `stackPath` and `networks`. Keep `containers` and `quadlet` — they are
still needed for SOPS `restartUnits` and `quadlet.alias` references.

```nix
# Before:
let
  stackPath = "/etc/stacks/service-name";
  my = config.services.my.service-name;
  inherit (config.virtualisation.quadlet) containers networks;
  inherit (flake.lib) quadlet;
in

# After:
let
  my = config.services.my.service-name;
  inherit (config.virtualisation.quadlet) containers;
  inherit (flake.lib) quadlet;
in
```

- Remove `stackPath` — use `my.stack.path` instead everywhere
- Remove `networks` — auto-wired by `stack.network.enable`

### 2. Delete manual boilerplate

Remove these sections entirely — they are auto-generated by the stack module:

- **`systemd.tmpfiles.rules`** — replaced by `stack.directories`
- **`virtualisation.quadlet.networks.<name>.networkConfig`** — replaced by
  `stack.network.enable = true`
- **`backup.paths = [ stackPath ]`** — auto-wired from `stack.path`
- **`backup.systemd.unit = ...`** — auto-wired from stack containers
- **`serviceConfig.Restart = "always"`** on every container — already the
  quadlet-nix default, remove all occurrences
- **`users.users` / `users.groups`** blocks — replaced by `stack.user`

### 3. Move containers into the stack

Move container definitions from `virtualisation.quadlet.containers` into
`services.my.<name>.stack.containers` using **short names**. The stack
auto-prefixes them (e.g. `web` → `karakeep-web`).

```nix
# Before:
virtualisation.quadlet.containers = {
  karakeep-web = {
    containerConfig = {
      image = "ghcr.io/karakeep-app/karakeep:0.31.0";
      publishPorts = [ "127.0.0.1:${toString my.port}:3000" ];
      networks = [ networks.karakeep.ref ];
      networkAliases = [ "web" ];
      ...
    };
    unitConfig = {
      After = [ containers.karakeep-meilisearch.ref ];
      Requires = [ containers.karakeep-meilisearch.ref ];
    };
    serviceConfig.Restart = "always";
  };
  karakeep-meilisearch = { ... };
};

# After:
services.my.karakeep.stack = {
  enable = true;
  network.enable = true;
  main = "web";
  internalPort = 3000;
  security.enable = true;

  containers = {
    web = {
      containerConfig = {
        image = "ghcr.io/karakeep-app/karakeep:0.31.0";
        # publishPorts auto-wired by stack.main + stack.internalPort
        # networks auto-wired by stack.network
        networkAliases = [ "web" ];
        ...
      };
      # replaces manual unitConfig.After/Requires
      dependsOn = [ "meilisearch" ];
    };
    meilisearch = { ... };
  };
};
```

What to remove from each container when moving:
- **`publishPorts`** on the main container — auto-wired by `stack.main` +
  `stack.internalPort`
- **`networks = [ networks.<name>.ref ]`** — auto-wired by `stack.network`
- **`serviceConfig.Restart = "always"`** — already the quadlet-nix default
- **`unitConfig.After/Requires`** for sibling containers — replaced by
  `dependsOn`

What to keep:
- **`networkAliases`** — still needed for inter-container DNS
- **`unitConfig.After/Requires`** for **non-sibling** dependencies (containers
  outside this stack)
- **`unitConfig."X-RestartTrigger"`** — still needed for config file restarts

### 4. Add stack configuration

```nix
services.my.service-name = {
  # keep existing: port, domain, proxy
  backup.enable = true;  # paths and units are auto-wired
  stack = {
    enable = true;
    directories = [ "data" "config" ];
    network.enable = true;  # only for multi-container stacks
    main = "web";           # short name of the main container
    internalPort = 3000;    # container-internal port
    security.enable = true; # OWASP hardening

    containers = {
      # ... moved from virtualisation.quadlet.containers
    };
  };
};
```

If the stack had custom users/groups, add:

```nix
stack.user = {
  enable = true;
  name = "myuser";     # defaults to service name
  uid = 1000;          # or null for auto-assigned
};
```

### 5. Update references throughout the file

#### Volume paths

Replace `${stackPath}/...` with `${my.stack.path}/...`:

```nix
# Before:
volumes = [ "${stackPath}/data:/data" ];
# After:
volumes = [ "${my.stack.path}/data:/data" ];
```

#### SOPS template restartUnits

Still use full prefixed names from `config.virtualisation.quadlet.containers`:

```nix
# The full prefixed names still work because the stack copies containers there
sops.templates."service.env" = {
  restartUnits = map quadlet.service [
    containers.service-name-web
    containers.service-name-db
  ];
  content = ''
    SECRET=${config.sops.placeholder.serviceSecret}
  '';
};
```

#### Inter-container hostnames

Use `quadlet.alias` with the full prefixed name:

```nix
# Before:
MEILI_ADDR = "http://meilisearch:7700";
# After:
MEILI_ADDR = "http://${quadlet.alias containers.karakeep-meilisearch}:7700";
```

### 6. Delete the virtualisation.quadlet block

Since containers are now in the stack, remove the entire
`virtualisation.quadlet` block (containers, networks, etc.):

```nix
# Before:
virtualisation.quadlet =
  let
    inherit (config.virtualisation.quadlet) networks;
  in
  {
    networks.service-name.networkConfig = { ... };
    containers = { ... };
  };

# After: nothing — containers are in the stack, network is auto-generated
```

## Special Cases

### Stacks with host networking (e.g. home-assistant)

Containers using `networks = [ "host" ]` cannot use stack network auto-wiring.
Define these containers in `virtualisation.quadlet.containers` directly (not in
`stack.containers`). The stack module can still handle directories and backup.

```nix
services.my.home-assistant = {
  port = 8123;
  backup.enable = true;
  stack = {
    enable = true;
    directories = [ "config" ];
    # no network, no containers — host networking is managed directly
  };
};

# Container stays in virtualisation.quadlet.containers
virtualisation.quadlet.containers.home-assistant = {
  containerConfig = {
    image = "ghcr.io/home-assistant/home-assistant:2026.2.3";
    networks = [ "host" ];
    ...
  };
};
```

### Stacks with custom directory permissions

```nix
# Before:
systemd.tmpfiles.rules = [
  "d ${stackPath}/data 0755 1000 1000"
  "d ${stackPath}/mariadb/config 0755 root root"
];

# After:
stack.directories = [
  { path = "data"; mode = "0755"; owner = "1000"; group = "1000"; }
  { path = "mariadb/config"; }  # defaults to 0755 root root (or 0750 user if stack.user enabled)
];
```

### Stacks with security exceptions

Per-container security overrides go directly on the container using `security.*`:

```nix
stack = {
  security.enable = true;  # global: all containers get hardening
  containers = {
    web = { ... };
    meilisearch = {
      containerConfig = { ... };
      # override readOnly for this container only
      security.readOnlyRootFilesystem = false;
    };
  };
};
```

For containers that need capabilities, add them directly — the stack module
uses `mkDefault` so explicit `containerConfig` values always win:

```nix
stack.containers.main = {
  containerConfig = {
    addCapabilities = [ "NET_ADMIN" "NET_RAW" ];
  };
};
```

### Stacks where the function argument list changes

If the stack no longer references `flake.lib.quadlet` or
`config.virtualisation.quadlet.containers` (rare — only if there are no SOPS
templates), both `flake` and `containers` can be removed from the let block.

## Checklist

For each stack file:

- [ ] Remove `stackPath` let binding, use `my.stack.path`
- [ ] Remove `networks` from inherit (auto-wired by `stack.network`)
- [ ] Delete `systemd.tmpfiles.rules`
- [ ] Delete `users.users` / `users.groups` blocks
- [ ] Delete manual `networks.<name>.networkConfig`
- [ ] Move containers from `virtualisation.quadlet.containers` into
      `stack.containers` using short names
- [ ] Remove `publishPorts` from main container (auto-wired by `stack.main`)
- [ ] Remove `networks = [ ... ]` from each container (auto-wired)
- [ ] Replace `unitConfig.After/Requires` with `dependsOn` for sibling deps
- [ ] Remove all `serviceConfig.Restart = "always"`
- [ ] Delete `backup.paths` and `backup.systemd.unit` (auto-wired)
- [ ] Add `stack = { enable = true; ... }` block with `main`, `internalPort`,
      `security.enable`, etc.
- [ ] Replace `${stackPath}` with `${my.stack.path}` in volumes
- [ ] Replace hardcoded inter-container hostnames with `quadlet.alias`
- [ ] Delete the `virtualisation.quadlet` block (if fully migrated)
- [ ] Verify the file has no LSP errors
